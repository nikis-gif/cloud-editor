How Cloud works

A short setup guide for connecting your workspace and using the editor without friction.

Connection flow

  1. Open the plugin inside your controlled game development environment.
  2. Create a private session and copy both generated values.
  3. Paste the Session ID and Hidden Secret on the Cloud start screen.
  4. After connecting, Cloud loads the project tree and keeps the workspace available for editing.

Daily workflow

  1. Use the Explorer to open scripts and keep folders organized.
  2. Use tabs when jumping between multiple scripts.
  3. Use split view when comparing or editing two scripts at the same time.
  4. Save often with Ctrl + S and use the footer status to confirm the editor state.

Output mirror

  1. Run the game while the Cloud session is active.
  2. Prints, warnings, and runtime errors are mirrored to the Output panel.
  3. Errors show the runtime source path and stack details when available.
  4. Drag the top edge of Output to resize the panel.

All shortcuts

Ctrl + SSave the active script.
Ctrl + WClose the active tab.
Ctrl + 1-9Switch to a tab by position.
Ctrl + ICreate a new item from the selected parent.
Ctrl + DDuplicate selected Explorer items or the active script.
Ctrl + CCopy selected Explorer items.
Ctrl + XCut selected Explorer items.
Ctrl + VPaste copied or cut items into the selected parent.
Shift + ClickSelect a range of scripts in the Explorer.
Ctrl + ClickToggle one item in the current selection.
DeleteRemove selected scripts or folders.
F2Rename the selected item.
Ctrl + Shift + EFold the active script.
Ctrl + EUnfold the active script.
Ctrl + +Increase the Cloud UI zoom.
Ctrl + -Decrease the Cloud UI zoom.
Ctrl + 0Reset the Cloud UI zoom.
EscClose modals, panels, Learn, search, or create dialogs.

Explorer actions

  1. Create scripts, local scripts, module scripts, and folders from valid parents.
  2. Copy, cut, paste, duplicate, rename, and delete directly from the Explorer.
  3. Use multi-select before copy, cut, duplicate, or delete actions.

Editor behavior

  1. Cloud keeps editor shortcuts separate from Explorer shortcuts when focus is inside code.
  2. Copy and cut text still work normally inside the editor.
  3. Explorer copy, cut, paste, and delete run when focus is outside the code editor.
